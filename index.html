📖 Google Ads API 연동 가이드
필수 항목(Client ID · Client Secret · Refresh Token · 고객 ID)을 순서대로 발급받으세요.
💡 Refresh Token 방식을 사용합니다. 한 번 발급하면 만료 걱정 없이 Access Token을 자동으로 갱신합니다.
Developer Token은 관리자(MCC) 계정에서만 발급 가능한 선택 항목입니다.
STEP 1 Client ID · Client Secret 발급 GCP 콘솔 →
  1. console.cloud.google.com 접속 → 프로젝트 선택(또는 새 프로젝트 생성)
  2. 좌측 메뉴 API 및 서비스 → 라이브러리에서 Google Ads API 검색 후 사용 설정
  3. API 및 서비스 → 사용자 인증 정보 → + 사용자 인증 정보 만들기 → OAuth 클라이언트 ID 클릭
  4. 애플리케이션 유형: 데스크톱 앱 선택 → 이름 입력 → 만들기
  5. 팝업에서 클라이언트 ID와 클라이언트 보안 비밀번호(Client Secret) 복사
⚠️ OAuth 동의 화면이 설정되지 않았다면 OAuth 동의 화면 메뉴에서 앱 정보 먼저 저장하세요. 유형은 외부로 선택해도 됩니다.
STEP 2 Refresh Token 발급 — 만료없음 OAuth Playground →
  1. OAuth 2.0 Playground 접속
  2. 우측 상단 ⚙️ 설정 클릭 → Use your own OAuth credentials 체크 → STEP 1에서 복사한 Client ID · Client Secret 입력
  3. 좌측 Step 1 스코프 입력란에 아래 값 입력 후 Add to Scopes 클릭:
    https://www.googleapis.com/auth/adwords
  4. Authorize APIs 클릭 → Google 계정 로그인 → 권한 허용
  5. Step 2 화면에서 Exchange authorization code for tokens 클릭
  6. Refresh Token 값 복사 (1//로 시작) → 등록 폼에 입력
✅ Refresh Token은 만료되지 않습니다. 이 토큰으로 Access Token을 자동 발급하므로 매번 재발급할 필요가 없습니다.
Access Token(ya29.로 시작)은 선택 입력이며, 비워두면 자동 처리됩니다.
STEP 3 고객 ID (Customer ID) 확인
  • Google Ads(ads.google.com) 로그인 → 화면 우측 상단에 123-456-7890 형식으로 표시됩니다.
  • 하이픈(-) 없이 숫자 10자리만 입력하거나, 그대로 복사해도 자동으로 처리됩니다.
  • 여러 광고주를 하나의 관리자(MCC) 계정으로 운영 중이라면 관리자 계정 ID도 입력하세요.
선택 Developer Token 관리자 계정만 해당 API 센터 →
  • 관리자(MCC) 계정에만 개발자 토큰이 존재합니다. 일반 광고주 계정에는 API 센터 메뉴가 보이지 않습니다.
  • 발급 경로: Google Ads → 도구 및 설정 → API 센터 → 개발자 토큰 확인
  • 일반 광고주 계정만 관리한다면 이 항목은 비워두세요.
⚠️ 처음 발급된 Developer Token은 테스트 계정만 접근 가능합니다. 실계정 접근을 위해서는 Google에 기본 접근(Basic Access) 승인 신청이 필요합니다.

🔵 Meta API 설정 가이드
시스템 사용자(System User) 방식으로 발급된 토큰은 만료 기간이 없습니다.
💡 시스템 사용자 토큰 방식을 사용합니다. 개인 Facebook 계정과 독립적으로 동작하며, 한 번 발급 후 만료 걱정 없이 계속 사용할 수 있습니다.
STEP 1 시스템 사용자 생성 시스템 사용자 →
  1. business.facebook.com/settings/system-users 접속
  2. 추가 클릭 → 이름 입력 → 역할: Admin (관리자) 선택 후 저장
💡 Admin 선택 시 비즈니스 내 전체 광고 계정에 접근할 수 있습니다.
STEP 2 광고 계정에 시스템 사용자 배정 광고 계정 설정 →
  1. business.facebook.com/settings/ad-accounts 접속
  2. 조회할 광고 계정 선택 → 사용자 추가 클릭 → 생성한 시스템 사용자 선택
  3. 권한 선택 후 저장

광고 계정 관리
광고 생성·수정까지 가능한 전체 권한 — 필요한 경우에만 선택
STEP 3 Meta 앱 준비 Meta for Developers →
  1. developers.facebook.com/apps 접속 → 앱 만들기 클릭
  2. 앱 유형: 비즈니스 선택 → 앱 이름 입력 후 저장
💡 이미 비즈니스 앱이 있다면 새로 만들지 않아도 됩니다.
STEP 4 영구 토큰 발급 — 만료없음 시스템 사용자 →
  1. 시스템 사용자 설정으로 돌아와 생성한 사용자 선택 → 토큰 생성 클릭
  2. STEP 3에서 만든 앱 선택 → 권한에서 ads_read, read_insights 체크
  3. 발급된 토큰 (EAA...) 복사 → Access Token 입력란에 붙여넣기
✅ 시스템 사용자 토큰은 만료되지 않습니다. 한 번 발급 후 재발급 없이 계속 사용할 수 있습니다.
STEP 5 광고 계정 ID 확인 광고 계정 설정 →
  • 광고 계정 설정에서 계정 ID를 확인합니다.
  • 숫자만 입력하거나 act_123456789 형태로 입력해도 자동 처리됩니다.
⚠️ 토큰은 광고 데이터 접근 권한을 가진 민감 정보입니다. 외부에 공유하지 마세요.
