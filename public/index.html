네이버 검색광고 API

네이버 광고 API 값은 광고주에게 문의해서 받을 수 있고, 광고주 계정에서 직접 확인할 수도 있습니다. 경로는 광고주센터 → 도구 → SA API 사용 관리입니다.

네이버 광고주센터SearchAd API 문서
1
광고주 계정 확인API는 실제 광고주 계정 기준으로 조회됩니다.
2
SA API 사용 관리도구 메뉴에서 Customer ID, Access License, Secret Key를 확인합니다.
3
대시보드 저장선택 광고주의 네이버 API 설정에 3개 값을 입력하고 저장합니다.
4
데이터 불러오기저장 후 조회 기간을 선택하고 데이터 불러오기를 실행합니다.
네이버 API는 관리자 계정 전체가 아니라 광고주 계정 기준으로 조회됩니다. 여러 광고주를 관리한다면 광고주별로 API 값을 저장하세요.

Meta Marketing API: 시스템 사용자 방식

만료 걱정을 줄이려면 개인 계정 토큰보다 비즈니스 설정 → 시스템 사용자에서 시스템 사용자 토큰을 발급하는 방식을 권장합니다.

시스템 사용자광고 계정 권한Meta Developers광고 관리자
1
비즈니스 선택Business Settings에서 사용할 비즈니스 포트폴리오를 선택합니다.
2
시스템 사용자 추가사용자 → 시스템 사용자 → 추가를 눌러 관리자 또는 직원 시스템 사용자를 만듭니다.
3
광고계정 자산 배정조회할 광고계정을 시스템 사용자에게 배정하고 캠페인 조회/광고 계정 관리 권한을 부여합니다.
4
토큰 생성ads_read, read_insights 권한을 포함해 토큰을 생성합니다. 소재 조회에는 ads_read 권한이 필요합니다.
5
일반/협력 계정 분리일반 Ad Account ID와 협력광고 Ad Account ID를 각각 입력합니다.
6
전환 기준 설정구매, 장바구니, 리드 등 기준을 선택하고 사용할 action_type을 체크합니다.
커스텀 이벤트는 픽셀/앱 이벤트 관리자에 등록된 이벤트 이름입니다. 예: offsite_conversion.fb_pixel_custom_event, app_custom_event.fb_mobile_purchase. 실제 내려오는 이름은 소재분석/전환 로그나 Meta 이벤트 관리자에서 확인해야 합니다.

Google Ads API: MCC 기준 권장

여러 광고주를 관리한다면 관리자 계정(MCC)에 OAuth 권한과 Developer Token을 묶고, 하위 고객 계정을 연결하는 구조가 가장 편합니다.

Google Ads API 사용 설정OAuth PlaygroundGoogle Ads API 센터Google Ads
1
GCP API 활성화Google Cloud Console에서 Google Ads API를 사용 설정합니다.
2
OAuth 클라이언트 생성Client ID와 Client Secret을 생성합니다.
3
Refresh Token 발급OAuth Playground에서 adwords scope로 권한을 승인합니다.
4
Developer Token 확인MCC 계정의 도구 및 설정 → API 센터에서 확인합니다.
5
MCC ID 저장관리자 계정 ID를 Login Customer ID에 입력합니다. 하위 계정 자동 탐색을 시도합니다.
6
필요 시 Customer ID 추가특정 하위 광고주만 조회하려면 Customer ID에 따로 입력합니다.
MCC 하나만 있어도 OAuth 권한으로 접근 가능한 하위 고객 계정을 자동 탐색할 수 있습니다. 다만 API 권한/계층 구조에 따라 특정 Customer ID를 직접 입력해야 할 수 있습니다.

Meta API 설정 가이드

시스템 사용자를 추가하고, 조회할 광고계정을 시스템 사용자에게 배정한 뒤 Access Token을 생성합니다. 일반 광고계정과 협력광고 계정을 분리해서 입력하면 각각 일반 actions/action_values, 공유 catalog_segment_actions/catalog_segment_value 기준으로 계산합니다.

시스템 사용자광고 계정 권한Meta Developers이벤트 관리자
1
시스템 사용자 생성Business Settings → 사용자 → 시스템 사용자에서 추가합니다.
2
광고계정 자산 추가조회할 광고계정을 시스템 사용자에게 배정합니다.
3
토큰 생성ads_read, read_insights 권한을 포함해 토큰을 발급합니다.
4
전환 기준 선택한글 기준을 선택하고 사용할 action_type을 체크합니다.
커스텀 이벤트 예시 offsite_conversion.fb_pixel_custom_event offsite_conversion.fb_pixel_회원가입완료 app_custom_event.fb_mobile_purchase

Meta Marketing API

일반/협력광고 + 전환 기준
일반 광고계정 ID를 줄바꿈/쉼표로 입력합니다. 일반 구매/리드/장바구니는 actions/action_values 기준입니다.
여기에 입력된 계정도 실제 조회 대상에 포함됩니다. 협력광고 계정은 catalog_segment_actions / catalog_segment_value 기준으로 계산합니다.
기준을 선택하면 아래 action_type 후보가 한글 라벨로 표시됩니다. 일반 광고계정은 actions/action_values 기준, 협력광고 계정은 catalog_segment_actions/catalog_segment_value 기준을 유지합니다.
사용 action_type 체크체크한 action_type만 API 산출 기준으로 저장됩니다. 일반 계정은 actions/action_values에서 선택값을 찾고, 공유 전환은 기존 catalog_segment 기준을 유지합니다.
커스텀 이벤트는 Meta 이벤트 관리자에서 이벤트 이름을 확인하거나, API 응답 actions 배열의 action_type 값을 보고 입력하세요. 여러 개는 줄바꿈·쉼표·세미콜론으로 구분합니다.

Google Ads API 설정 가이드

관리자 계정(MCC) 하나에 OAuth/Developer Token을 설정해두면 하위 광고계정 자동 탐색을 시도합니다. 특정 하위 광고주만 조회해야 할 때만 Customer ID를 추가 입력하세요.

Google Ads API 활성화OAuth PlaygroundAPI 센터
1
GCP API 활성화Google Ads API를 사용 설정합니다.
2
OAuth 정보 입력Client ID, Secret, Refresh Token을 입력합니다.
3
MCC ID 입력관리자 계정 ID를 MCC Login Customer ID에 입력합니다.
4
Customer ID 선택비워두면 접근 가능한 하위 계정 자동 조회를 시도합니다.
Google Ads API는 Developer Token이 필요합니다. Developer Token은 보통 관리자 계정의 API 센터에서 확인합니다.
